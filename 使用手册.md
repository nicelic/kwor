# kwor 使用手册

> 面向第一次接触本项目的新手、小白、非技术用户。
>
> 这份文档重点讲“怎么用”“先做什么”“改完会影响什么”，不是源码讲解。
>
> English version: [User Manual.md](./User%20Manual.md)

---

## 1. 先用一句人话理解这个项目

`kwor` 是一个带网页后台的代理管理面板。

你可以把它理解成：

> **用网页去管理节点入口、用户账号、订阅链接、TLS 证书，以及一些系统功能的控制台。**

它不是只有一个简单页面，而是把这些东西放到了一起：

- 面板登录后台
- sing-box 配置链
- mihomo 配置链
- 订阅分发
- TLS / 证书
- 流量统计
- 防火墙
- 端口转发
- 反向代理
- 系统优化
- 本机监控

所以第一次使用时，不要想着“一口气把所有页都配完”，那样最容易乱。

---

## 2. 使用前先记住 8 个最重要的概念

### 2.1 面板地址和订阅地址不是一回事

- **面板地址**：你自己登录后台的地址
- **订阅地址**：给客户端导入节点、拉取配置的地址

它们可以不同：

- 面板有自己的端口、路径、域名
- 订阅也有自己的端口、路径、域名

所以：

- 改了面板地址，不代表订阅地址也会变
- 改了订阅地址，也不代表后台登录地址会变

---

### 2.2 入站就是“入口”

入站可以理解成：

> 别人从哪个协议、哪个端口、带什么 TLS 方式连进来。

比如：

- 443 端口上的 VLESS
- 8443 端口上的 Hysteria2
- 某个端口上的 Socks / HTTP / Mixed

---

### 2.3 用户就是“账号”

用户页主要管：

- 用户名
- UUID / 密码 / 密钥等身份信息
- 流量总量
- 到期时间
- 用户分组
- 绑定到哪些入站

简单理解：

- 入站决定“门怎么开”
- 用户决定“谁能进、能用多久、能用多少”

---

### 2.4 出站就是“出口”

出站决定：

> 这些流量出去的时候，往哪里走、通过谁走。

路由规则、DNS 规则，都会和出站联动。

---

### 2.5 sing-box 和 mihomo 是两条并行链

左侧菜单里你会看到两组很像的页面：

- sing-box：入站、用户、出站、TLS、路由、DNS
- mihomo：入站、用户、出站、TLS、路由、DNS

这不是装饰性的重复页面，而是两套独立链路。

新手最重要建议：

- 第一天只先选一条链用
- 要么先只做 sing-box
- 要么先只做 mihomo
- 不要一开始两边一起改

---

### 2.6 TLS / HTTPS 就是“加密访问”

TLS 会影响两类地址：

- 面板是不是用 `https://`
- 订阅是不是用 `https://`

如果你暂时没有域名、没有正式证书，也可以先用 HTTP 跑起来，后面再补证书。

---

### 2.7 设置页不是一个简单表单

这个项目的“设置”页面非常重要，但也最容易误解。

它不是“所有内容都点同一个保存”。

你要先记住：

#### 第一组：顶部统一保存

这些标签页改完以后，要点页面上方统一的“保存”按钮：

- 界面
- 订阅
- JSON 订阅
- Clash 订阅
- Language

#### 第二组：各页面自己保存

这些标签页通常各自有自己的按钮、开关、保存逻辑：

- 流量管理
- 防火墙
- 转发
- 优化
- 证书管理
- 反向代理
- 内核管理
- 监控

也就是说：

- 你在前 5 个标签里改东西，要看页面顶部
- 你在后 8 个标签里改东西，要看当前卡片里的按钮

---

### 2.8 有些设置不是“保存一下”这么简单

有些页面改完，影响的是整台机器，不只是某个节点：

- 改 SSH 端口会重启 SSH
- 开防火墙会重建规则链
- 改系统 DNS 会重写 `resolv.conf`
- 改 sysctl 会立刻应用系统参数
- 改 MTU 会生成脚本并尝试常驻
- 装 / 删 ACME、vnstat、nftables 会动系统环境

所以新手要建立一个习惯：

> **每次只改一小块，改完立刻验证。**

## 3. 第一次使用时，最推荐的顺序

### 3.1 第一步：先让面板能正常打开

你先要解决的是：

- 能不能打开后台
- 能不能登录
- 登录后会不会跳错地址

常见默认值通常是：

- 面板端口：`8888`
- 面板路径：`/app/`
- 订阅端口：`22780`

如果是首次初始化，订阅路径通常会自动生成一个随机值。

---

### 3.2 第二步：登录后先看主页

主页不是摆设，它是你的总控台。

你在这里先确认：

1. 面板本身有没有正常运行
2. sing-box 有没有运行
3. mihomo 有没有运行
4. 系统信息有没有明显异常
5. 日志和备份入口是否正常

主页上还能直接做这些动作：

- 启动 sing-box
- 停止 sing-box
- 重启 sing-box
- 启动 mihomo
- 停止 mihomo
- 重启 mihomo
- 打开日志
- 打开备份恢复

新手建议：

- 配置出问题时，先回主页看核心是否真的在运行
- 不要一上来先去高级页，把主页看懂很重要

---

### 3.3 第三步：只先改最基础的设置

第一次登录后，先只改这几个地方：

1. `设置 -> 界面`
2. `设置 -> 订阅`
3. `设置 -> Language`

先把地址、端口、语言这些最基础的东西理顺。

---

### 3.4 第四步：再开始做业务配置

如果你用的是 sing-box，推荐顺序：

1. TLS 设置
2. 入站管理
3. 用户管理
4. 出站管理
5. 路由列表
6. DNS
7. 订阅管理

如果你用的是 mihomo，推荐顺序：

1. mihomo TLS 设置
2. mihomo 入站管理
3. mihomo 用户管理
4. mihomo 出站管理
5. mihomo 路由列表
6. mihomo DNS
7. 订阅管理

---

### 3.5 第五步：最后再碰高级系统功能

这些页面先不要急着动：

- 流量管理
- 防火墙
- 转发
- 优化
- 证书管理里的高级项
- 反向代理
- 内核管理
- 监控保留策略

不是说它们不重要，而是它们一旦配错，影响的就不只是某个用户或某个节点。

## 4. 左侧菜单怎么理解

### 4.1 主页

作用：

- 看整体状态
- 看系统图表
- 启停核心
- 看日志
- 做备份

适合谁：

- 所有人

---

### 4.2 订阅管理

作用：

- 整理节点分发方式
- 生成订阅
- 生成二维码
- 做分组

适合谁：

- 想给客户端统一导入链接的人

注意：

- 它不是创建入站的地方
- 它也不是创建用户账号的地方

---

### 4.3 入站管理

作用：

- 开协议入口
- 定端口
- 定 TLS
- 看端口状态
- 看端口日志

一句话理解：

> 在服务器上开一扇门，让别人从这扇门进来。

---

### 4.4 用户管理

作用：

- 建账号
- 批量建账号
- 设流量
- 设到期时间
- 绑定入站
- 看二维码和流量图

一句话理解：

> 给不同的人发不同的通行证。

---

### 4.5 出站管理

作用：

- 定义流量出口
- 给出口分组
- 看出口在线情况

一句话理解：

> 流量出门以后往哪走。

---

### 4.6 路由列表

作用：

- 决定什么流量走哪个出站
- 决定直连、代理、阻断、分流

一句话理解：

> 交警规则。

---

### 4.7 DNS

作用：

- 管域名解析
- 配 DNS 服务器
- 配 DNS 规则

一句话理解：

> 决定域名怎么查、查谁、什么时候走直连 DNS，什么时候走代理 DNS。

---

### 4.8 基础信息

作用：

- 日志设置
- NTP
- 实验项
- Clash API / V2Ray API 一些高级项

特点：

- 不是新手必改页
- 更偏底层运行参数

---

### 4.9 管理员

作用：

- 管后台账号
- 改密码
- 看登录记录
- 看更改记录
- 管 API Token

新手第一建议：

- 第一次跑起来后，尽快改默认密码

---

### 4.10 Settings

作用：

- 管全局设置
- 管高级系统功能

特点：

- 这是最容易把项目改“活”也最容易改“断”的页面

---

## 5. 新手最常走的基础流程

## 5.1 如果你只想先做一个能用的 sing-box 账号

按这个顺序：

1. 主页确认程序正常
2. 设置里确认面板和订阅地址
3. TLS 设置里准备证书或 TLS 配置
4. 入站管理新建一个入站
5. 用户管理新建一个用户
6. 把这个用户绑定到刚才那个入站
7. 去用户页看二维码或链接
8. 需要统一导入时，再去订阅管理

---

## 5.2 如果你只想先做一个能用的 mihomo 账号

按这个顺序：

1. 主页确认程序正常
2. 设置里确认面板和订阅地址
3. mihomo TLS 设置
4. mihomo 入站管理
5. mihomo 用户管理
6. mihomo 出站管理
7. mihomo 路由
8. mihomo DNS
9. 订阅管理

特别提醒：

`mihomo DNS` 页面主要是在配置 mihomo 自己的 DNS 行为，不是额外给整个系统开一个新的本地 DNS 服务。

## 6. 设置页详细说明

这是这份手册里最重要的一章。

---

## 6.1 设置 -> 界面

这一页是“后台自己怎么被访问”。

改完后，通常要点页面顶部的统一“保存”。

### 地址 `webListen`

它是**面板监听地址**，不是你发给别人看的域名。

常见理解：

- 留空：通常按默认方式监听
- `127.0.0.1`：只允许本机访问
- `0.0.0.0`：允许外部访问

新手建议：

- 不懂就先别乱改

---

### 端口 `webPort`

这是面板登录端口。

比如：

- `8888`
- `443`

改完后会发生什么：

- 旧端口访问会失效
- 你需要用新端口重新打开后台

新手建议：

- 先用默认端口更稳
- 改到 `80` 或 `443` 前，先确认端口没被其他程序占用

---

### 面板路径 `webPath`

这是后台 URL 路径。

常见例子：

- `/app/`
- `/panel/`

改完后会发生什么：

- 原地址会失效
- 需要用新路径打开

新手建议：

- 最稳妥的是保持 `/app/`
- 如果要改，尽量保留前后斜杠

---

### 域名 `webDomain`

这是面板展示和拼接地址时更适合使用的域名。

适合什么时候填：

- 你有正式域名
- 你准备上 HTTPS
- 你不想一直显示 IP 地址

不要混淆：

- `webListen` 是监听谁
- `webDomain` 是展示成什么

---

### 面板 URI `webURI`

这是完整面板地址的辅助字段。

新手建议：

- 不懂就留空
- 先让系统按端口、路径、域名自动拼

---

### 会话超时 `sessionAge`

它影响登录保持多久。

你可以这样理解：

- 小：更容易掉登录
- 大：登录更持久

---

### 流量过期时限 `trafficAge`

它影响一些流量历史保留时间。

你可以理解成：

- 时间越长，可看历史越多
- 数据也会积累更多

---

### 时区 `timeLocation`

这项很重要，因为它会影响：

- 到期时间显示
- 图表时间
- 日志时间
- 某些定时行为的时间感知

中国大陆常见可用值：

- `Asia/Shanghai`

---

### 版本检查 / 安装区域

这一块可以：

- 看本地版本
- 看远程版本
- 检查更新
- 选择版本
- 安装版本

新手建议：

- 如果你是手工部署，理解后可以使用
- 如果你是 Docker 或特殊环境，不要想当然直接点“安装”
- 按钮不可用，通常说明当前部署方式不适合面板内升级

---

## 6.2 设置 -> 订阅

这一页是“客户端如何拿到订阅”。

改完后，也要点页面顶部统一“保存”。

### Base64 编码 `subEncode`

很多订阅客户端会直接读取 Base64 编码内容。

新手建议：

- 不确定时保持开启

---

### 显示用户信息 `subShowInfo`

通常影响订阅里是否附带流量、到期等提示。

适合什么时候开：

- 你希望客户端端看到更多账号信息

---

### 地址 `subListen`

这是**订阅服务监听地址**。

它和面板监听地址逻辑类似，但对象不同。

---

### 端口 `subPort`

这是订阅服务端口。

常见默认值通常是：

- `22780`

改完会发生什么：

- 旧订阅链接很可能失效
- 二维码也会失效
- 客户端要改地址

---

### 域名 `subDomain`

这是订阅对外更适合展示的域名。

适合什么时候填：

- 你想发域名订阅而不是 IP
- 你给订阅开了 HTTPS

---

### 路径 `subPath`

这是订阅路径。

首次初始化时通常会自动生成随机路径。

强烈建议：

- 不要改得过于简单
- 不要随手改成 `/sub/` 这种很容易被猜的值

改完会发生什么：

- 旧链接和旧二维码可能全部失效

---

### 自动更新时间 `subUpdates`

它更像是给客户端的“建议更新周期”。

新手建议：

- 保持默认即可
- 没必要设得太频繁

---

### 订阅 URI `subURI`

它和面板 URI 类似，是完整订阅地址的辅助项。

新手建议：

- 不懂就留空

---

## 6.3 设置 -> JSON 订阅

这页不是“基础能不能用”的第一层，而是：

> 生成 sing-box 风格 JSON 订阅时，用什么模板和高级规则。

改完后要点顶部统一“保存”。

页面里还有一个单独按钮：

- `重置 JSON 订阅`

它只会重置这一页的模板内容，不影响其他设置页。

### 这页适合谁

- 已经能正常使用
- 现在想自定义下发给客户端的 JSON 结构

### 新手建议

- 如果你还没跑通基础链路，先不要动这页

### 你在这里大概会碰到什么

- 证书库选择
- 日志模板
- DNS 模板
- fakeip
- bootstrap DNS
- inbound / TUN 模板
- 路由规则集来源
- 延迟测试相关选项
- sniff / hijack-dns / external controller 等高级项

### 这页最容易造成的问题

- 节点没完全坏，但客户端行为变怪
- DNS 看起来“能用”，但部分站点打不开
- 下发的 JSON 配置和你预期不一致

---

## 6.4 设置 -> Clash 订阅

这一页和上面的 JSON 订阅类似，但目标更偏：

- Clash 风格
- Mihomo 风格

改完后也要点顶部统一“保存”。

页面里也有一个单独按钮：

- `重置 CLASH 订阅`

它只重置这一页模板，不影响其他设置。

### 你在这里会看到什么

- mixed-port
- allow-lan
- external-controller
- unified-delay
- tcp-concurrent
- TUN 相关
- fake-ip / fallback / nameserver
- 规则集模板
- sniffer
- hosts
- UDP 屏蔽类高级项

### 新手建议

- 如果你还没有把基础的出站、路由、DNS 理顺，这页先不要深改

### 最容易踩的坑

- 不是完全打不开，而是“行为异常”
- 比如局域网暴露、Fake-IP 表现异常、DNS 解析逻辑不对

---

## 6.5 设置 -> Language

这页最简单：

- 切换界面语言

新手建议：

- 先切成你最熟悉的语言，再继续配项目

---

## 6.6 设置 -> 流量管理

这一页不是顶部统一保存，而是页面内部自己保存。

它主要管：

- vnstat 安装与删除
- 流量统计开关
- 月流量上限
- 流量重置日
- 重置当前流量
- 重置总流量

### 你在这里会看到什么

- 当前统计状态
- 当前系统是否装了 vnstat
- 版本、路径、数据目录
- 当前累计上行、下行、总流量
- 月流量使用比例

### 重要按钮

- 下载 / 删除 vnstat
- 开关流量统计
- 保存流量设置
- 重置流量
- 重置总流量

### 新手要知道的影响

- 关掉流量统计期间，新流量通常不会计入面板统计
- 删除 vnstat 不是只删一个页面数据，而是可能清理系统中的相关统计文件

### 适合什么时候用

- 你需要看服务器总流量
- 你需要按月重置统计

---

## 6.7 设置 -> 防火墙

这是高风险页面。

这页不是顶部统一保存，而是每个动作单独生效。

### 这一页能做什么

- 安装 nftables
- 开关整个面板托管防火墙
- 改 SSH 端口
- 开关 SSH 代理能力
- 管系统保留端口
- 新建普通规则
- 新建 GeoIP 规则
- 配 GeoIP 更新周期
- 立即热更新 GeoIP

### 这一页最关键的 4 件事

1. 面板端口要不要放行
2. SSH 端口要不要放行
3. 订阅端口要不要放行
4. GeoIP 会不会误伤自己

### 普通规则大概管什么

- 协议
- 端口 / 端口段
- IPv4 / IPv6 / 双栈
- 来源 IP / CIDR

### GeoIP 规则大概管什么

- 哪些国家放行
- 哪些国家阻断
- 哪些端口应用 GeoIP 识别
- 规则集来源和更新

### 非常重要的提醒

- 开防火墙前，先确认 SSH 和面板端口不会把自己挡在外面
- 改 SSH 端口后会重启 SSH 服务
- 最好保留一个现有登录会话，不要一次性断自己后路

---

## 6.8 设置 -> 转发

这一页用来做端口转发。

它也是页面内部自己保存，不走顶部统一保存。

### 适合什么场景

- 把本地端口转发到另一个 IP / 端口
- 做简单入口映射
- 做多端口或端口段转发

### 你会配置什么

- 协议
- IPv4 / IPv6
- 本地端口
- 目标 IP
- 目标端口
- 限速
- 单端口 / 范围 / 多端口模式

### 新手建议

- 先只做一条最简单的规则
- 先确认目标地址本身是通的
- 如果转发不通，不一定是面板规则错，也可能是目标服务本来就没开

---

## 6.9 设置 -> 优化

这是系统级页面，不是“性能小优化”那么简单。

它也是页面内部各自操作。

### 这页分成 4 块

#### 1. 禁用系统日志

会涉及：

- systemd journal 持久日志
- 配置文件重建
- journald 重启

适合什么时候用：

- 你明确知道自己为什么要减少系统日志写入

#### 2. sysctl 参数优化

会涉及：

- `/etc/sysctl.conf`
- `/etc/sysctl.d/...`
- 系统参数立即应用

适合什么时候用：

- 你明确知道内核网络参数要怎么调

#### 3. Linux DNS

会涉及：

- `resolv.conf`
- nameserver 重写

适合什么时候用：

- 你明确知道系统本机 DNS 要怎么设

#### 4. MTU 优化

会涉及：

- 当前默认网卡 MTU
- 生成脚本
- systemd 自启动

适合什么时候用：

- 你明确知道 MTU 对你当前线路有帮助

### 这页的总建议

- 不懂就不要动
- 因为改错后，影响的是整台机器网络，不是一个小功能

---

## 6.10 设置 -> 证书管理

这是整个项目里非常强的一页。

它也是各模块自己操作，不走顶部统一保存。

### 这页能做什么

- 下载 / 删除 acme.sh
- 申请正式证书
- 创建自签证书
- 管 ACME 账号
- 管 DNS 账号
- 查看证书列表
- 续签
- 强制续签
- 开 / 关自动续签
- 推送证书到目录
- 应用到面板
- 应用到订阅
- 看签发日志

### 新手最常用的 3 个场景

#### 场景 1：我有域名，想申请正式证书

最推荐方式：

- DNS 验证

前提：

- 你能控制域名 DNS
- 你有对应 DNS 提供商账号或 API 信息

#### 场景 2：我没有 DNS API，但 80 端口能用

可以考虑：

- HTTP standalone
- HTTP webroot

#### 场景 3：我只是先测试 HTTPS

可以先用：

- 自签证书

### 特别要分清的 3 件事

1. 证书“签出来了”不等于已经应用
2. 面板 HTTPS 和订阅 HTTPS 可以分别应用
3. 自签证书能用，不代表浏览器会信任

### 新手非常实用的建议

- 公网正式用：优先正式证书
- 临时测试：自签也行
- 应用证书以后，再回去检查界面地址和订阅地址是否正确

---

## 6.11 设置 -> 反向代理

这页也是页面内单独保存。

它的用途是：

> 把外面来的请求，按协议、端口、域名、路径，转发到你的目标服务。

### 你在这里通常会配什么

- 监听协议
- 监听端口
- Hosts
- Path Prefix
- 目标协议
- 目标地址
- 目标端口
- 目标路径
- IP 策略
- HTTP 版本策略
- 上游 TLS 校验
- 绑定证书

### 新手最容易搞混的两边

#### 监听侧

就是：

- 外面的人怎么访问你

#### 目标侧

就是：

- 你最终把请求转发到哪里

### 适合什么场景

- 你想把一个域名入口转到内部网站
- 你想用路径分流不同服务
- 你想统一入口

### 最重要的提醒

- 不要把流量又转回自己，形成死循环
- HTTPS 监听时记得证书要绑定对
- 路径前缀和目标路径要分清楚

- 清理旧内核包

### 适合谁

### 新手建议

## 6.13 设置 -> 监控

这一页是本机资源监控。

它也是页面内部自己保存。

### 这页能看什么

- CPU
- 内存
- 磁盘读写
- 网卡上下行
- 历史曲线
- 当前采样情况
- 监控数据库大小

### 这页能改什么

- 采样间隔
- 短期保留
- 归档保留
- 清空监控历史

### 你要这样理解

#### 采样间隔

- 多久采一次

#### 短期保留

- 分钟级、小时级历史保留多久

#### 归档保留

- 更长时间的粗粒度历史保留多久

### 清空监控历史会发生什么

- 会删掉监控曲线历史
- 会清理监控数据库中的历史统计
- 不会删主数据库
- 不会删面板基本设置

### 新手建议

- 不需要非常细的历史时，不要把采样设得太激进

## 8. 最容易踩的坑

## 8.1 改了面板端口或路径，原地址为什么打不开

因为你改的是入口本身。

例如：

- `8888` 改成 `9999`
- `/app/` 改成 `/panel/`

那旧地址当然会失效。

---

## 8.2 面板能打开，但订阅突然失效

因为面板地址和订阅地址是两套东西。

重点检查：

- `subPort`
- `subPath`
- `subDomain`
- `subURI`

---

## 8.3 防火墙一开，自己进不去了

常见原因：

- 面板端口没放行
- SSH 端口没放行
- 订阅端口没放行
- GeoIP 误伤了自己

---

## 8.4 开了 HTTPS，浏览器还报不安全

如果你用的是自签证书，这是正常现象。

不是项目坏了，而是浏览器不信任这张证书。

---

## 8.5 高级设置越改越乱

因为这些页面本来就不适合第一天深改：

- JSON 订阅
- Clash 订阅
- 防火墙
- 优化
- 内核管理

最稳的方法永远是：

1. 先跑通基础链路
2. 再碰高级模板
3. 再碰系统功能

---

## 8.6 Docker 环境里不要随便点面板内安装

因为很多 Docker 部署的正确升级方式是：

- 拉新镜像
- 重建容器

而不是在页面里直接替换程序文件。

先只做 sing-box，或者先只做 mihomo。

- 优化

这个项目功能很多，不等于第一天就必须全开。
